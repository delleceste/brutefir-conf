# Filters and DRC Operation

This document describes how filter files are organized and how `drc.sh` selects
geometry, sample rate, MPD output mode, and filter variants.

It is intentionally explicit so an AI agent can reason about the repository
without guessing path conventions.

## Core Concepts

The playback chain has four coordinated parts:

1. `filters/`: raw FIR filter coefficients and source/intermediate files.
2. `configs/<position>/brutefir-<rate><variant>.conf`: BruteFIR runtime
   configurations.
3. `drc.sh`: orchestration script that starts/stops BruteFIR and `virtual_oss`
   and selects the MPD output.
4. `mpd/musicpd.conf`: MPD output definitions.

The BruteFIR config is the authoritative link between a DRC mode and the raw
filter files. `drc.sh` does not infer filter paths from `filters/` directly. It
builds a config path and BruteFIR reads the filter paths from that config.

### `filters/120.blue`

`filters/120.blue` is the current structured filter set.

Numeric subdirectories are sample-rate-specific BruteFIR coefficient sets. Each
numeric directory is intended to contain:

```text
L.raw
R.raw
sox.txt
```

`L.raw` and `R.raw` are raw little-endian 64-bit float FIR coefficients
(`FLOAT64_LE`) for that exact sample rate.

`sox.txt` is the conversion log generated by `scripts/REW2raw-all-rates.sh`.
It records the source WAVs, SoX command lines, sample-rate FIR gain, and measured
stats.

`filters/120.blue/192000` contains the default 192 kHz filter pair:

```text
filters/120.blue/192000/L.raw
filters/120.blue/192000/R.raw
```

`filters/120.blue/192000/+2dB` contains the 192 kHz `+2dB` variant:

```text
filters/120.blue/192000/+2dB/L.raw
filters/120.blue/192000/+2dB/R.raw
```

The `rew/` directory contains original or source files exported by Room EQ
Wizard (REW), plus variant-specific source material. These files are not read by
BruteFIR directly.

### `filters/185-green`

`filters/185-green` is a candidate/source filter library for the 185 cm speaker
position. It currently contains named 192 kHz raw files rather than the
normalized runtime layout:

```text
filters/185/192000/L.raw
filters/185/192000/R.raw
```

The current BruteFIR config for that position is:

```text
configs/185/brutefir-192000.conf
```

That config currently references:

```text
filters/185/192000/L.raw
filters/185/192000/R.raw
```

Those runtime files/directories are not present in the current tree. The files
under `filters/185-green/` are candidates that must be selected/copied/converted
into the runtime paths referenced by `configs/185/brutefir-192000.conf`, or the
config must be changed to point at the chosen files.

Important naming distinction:

- `185-green` is the filter-set/source directory name.
- `185` is the current `drc.sh` position token because the config directory is
  `configs/185`.

If the intended runtime command is `./drc.sh 185-green 192000`, then there must
be a matching config file under:

```text
configs/185-green/brutefir-192000.conf
```

That directory does not currently exist.

## BruteFIR Config Directory Structure

`drc.sh` constructs the BruteFIR config path as:

```text
configs/<position>/brutefir-<actual_rate><variant>.conf
```

Where:

- `<position>` is the first positional argument passed to `drc.sh`.
- `<actual_rate>` is the requested rate, except for `resamp`, where it becomes
  `192000`.
- `<variant>` is optional and is appended directly to the filename, e.g. `+2dB`.

Examples:

```text
./drc.sh 120.blue 44100
  -> configs/120.blue/brutefir-44100.conf

./drc.sh 120.blue 192000
  -> configs/120.blue/brutefir-192000.conf

./drc.sh 120.blue 192000 +2dB
  -> configs/120.blue/brutefir-192000+2dB.conf

./drc.sh 120.blue resamp
  -> configs/120.blue/brutefir-192000.conf

./drc.sh 185 192000
  -> configs/185/brutefir-192000.conf
```

The config file then declares:

- `sampling_rate`
- left and right coefficient files
- coefficient format, usually `FLOAT64_LE`
- attenuation
- input/output wiring

## `drc.sh` Modes

Usage:

```sh
./drc.sh <position> <rate> [variant]
./drc.sh <position> resamp [variant]
./drc.sh off
```

Supported native rates are currently:

```text
44100
48000
88200
96000
192000
```

### Native Rate Mode

Native mode is used when the source track sample rate is known and the matching
BruteFIR filter/config exists.

Example:

```sh
./drc.sh 120.blue 96000
```

This does the following:

1. Stops any existing `brutefir` process.
2. Validates that this config exists:

   ```text
   configs/120.blue/brutefir-96000.conf
   ```

3. Stops any existing `virtual_oss` process tracked by `/tmp/virtual_oss.pid`.
4. Starts `virtual_oss` at `96000` Hz using:

   ```text
   /dev/dsp.play
   /dev/dsp.loop
   ```

5. Starts BruteFIR with the selected config.
6. Disables all MPD outputs.
7. Enables MPD output `DRC-native`.
8. Writes the requested state to `last_arg`.

In native mode MPD must not resample. `mpd/musicpd.conf` therefore uses:

```conf
format "*:*:*"
```

for `DRC-native`. MPD's `format` syntax is:

```text
sample_rate:bits:channels
```

An asterisk means that attribute is not enforced. `*:*:*` means:

- do not force sample rate
- do not force bit depth
- do not force channel count

The user is responsible for selecting the `drc.sh` rate that matches the source
track rate. For example:

- 44.1 kHz / 16-bit source -> `./drc.sh 120.blue 44100`
- 44.1 kHz / 24-bit source -> `./drc.sh 120.blue 44100`
- 96 kHz source -> `./drc.sh 120.blue 96000`

BruteFIR processes internally in floating point. Source bit depth is not the DRC
routing selector; sample rate is.

### `resamp` Mode

`resamp` mode is used when MPD should resample all source material to 192 kHz.

Example:

```sh
./drc.sh 120.blue resamp
```

This does the following:

1. Sets `actual_rate=192000`.
2. Selects:

   ```text
   configs/120.blue/brutefir-192000.conf
   ```

3. Starts `virtual_oss` at `192000` Hz.
4. Starts BruteFIR at `192000` Hz.
5. Enables MPD output `DRC-resamp`.

`DRC-resamp` in `mpd/musicpd.conf` intentionally contains:

```conf
format "192000:24:2"
```

That MPD output is the explicit resampling path. It is useful for mixed-rate
playlists, but it is not native-rate playback.

### Variant Mode

The optional third argument selects a filter/config variant by appending the
variant string to the BruteFIR config filename.

Example:

```sh
./drc.sh 120.blue 192000 +2dB
```

This selects:

```text
configs/120.blue/brutefir-192000+2dB.conf
```

That config points to:

```text
filters/120.blue/192000/+2dB/L.raw
filters/120.blue/192000/+2dB/R.raw
```

Variant names are not discovered automatically. A variant works only if the
matching config file exists.

### `off` Mode

`off` disables DRC:

```sh
./drc.sh off
```

This does the following:

1. Stops any existing `brutefir` process.
2. Enables only MPD output `1`, the direct DAC output.
3. Stops `virtual_oss` if `/tmp/virtual_oss.pid` exists.
4. Writes `off` to `last_arg`.

No BruteFIR config is required in `off` mode.

## MPD Outputs

Current relevant outputs:

```text
OKTO-DAC    direct DAC output, no DRC
DRC-native  native-rate DRC loopback, no MPD format conversion
DRC-resamp  DRC loopback with MPD-forced 192000:24:2 conversion
```

`drc.sh off` enables the direct DAC path by output number:

```sh
mpc enable only 1
```

Native DRC mode enables:

```sh
mpc enable DRC-native
```

Resampling DRC mode enables:

```sh
mpc enable DRC-resamp
```

## Filter Generation

For the `120.blue` layout, source REW WAV files are stored under:

```text
filters/120.blue/rew/
```

The script:

```sh
scripts/REW2raw-all-rates.sh
```

generates `L.raw`, `R.raw`, and `sox.txt` for every numeric rate directory under
a filter root.

Example:

```sh
scripts/REW2raw-all-rates.sh \
  -L filters/120.blue/rew/FLX-trimmed-48k.wav \
  -R filters/120.blue/rew/FRX-trimmed-48k.wav \
  -o filters/120.blue
```

Without `-y`, it asks before writing each rate directory. If `L.raw` or `R.raw`
already exists, it explicitly lists the existing files and asks for overwrite
confirmation.

Generated raw files are `FLOAT64_LE`.

The conversion does not peak-normalize. FIR coefficient scale is based on:

```text
scale = input_sample_rate / target_sample_rate
gain_db = 20 * log10(scale)
```

Peak values in `sox.txt` are diagnostics only.

## Rules for AI Agents

When changing this repository:

1. Do not assume a `filters/<position>` directory exists just because
   `configs/<position>` exists. Read the BruteFIR config to find the actual raw
   file paths.
2. Do not assume `185-green` is a valid `drc.sh` position argument. Currently
   `configs/185` exists, so the current command token is `185`.
3. Do not generate or overwrite `filters/120.blue/192000/L.raw` or `R.raw`
   unless explicitly asked. These are the current 192 kHz default filters.
4. In native playback, do not add fixed MPD output formats such as
   `44100:24:2`. They force MPD conversion. Use `DRC-native` with `*:*:*`.
5. Use `DRC-resamp` only when the user explicitly wants MPD to resample to
   192 kHz.
6. For a new filter rate, create all three pieces:

   ```text
   filters/<filter-root>/<rate>/L.raw
   filters/<filter-root>/<rate>/R.raw
   configs/<position>/brutefir-<rate>.conf
   ```

   and ensure the config points to the intended raw files.
7. For a new variant, create a config named:

   ```text
   configs/<position>/brutefir-<rate><variant>.conf
   ```

   and point it to the variant raw files.
8. After generating or changing filters, run `scripts/headroom_calc.py` or
   otherwise verify the required BruteFIR `attenuation`.
